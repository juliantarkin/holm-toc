# Dual-Agent Document Review v0.1

## Status
Draft workflow specification for later n8n implementation.

## Intent
Coordinate two agent edit proposals for one document without overwriting the source.

## API contracts
The OpenAI side of this workflow should use the OpenAI `Responses` API at `POST /v1/responses` with an OpenAI credential stored in n8n as `openai-toc`. For Phase 01, the recommended model is `gpt-5.1` for document editing and structured output; this is an inference from the current OpenAI model guidance, which recommends the latest GPT-5 family with the Responses API for reasoning and agentic tasks. If later phases need coding-specialized behavior, that should be a separate model decision rather than being implied by the word "Codex."

The Claude side should use the existing Anthropic execution pattern already used in your n8n environment, with its credential name documented in the eventual implementation workflow.

## Workflow steps
1. Trigger
2. Validate input payload
3. Read source document
4. Save source snapshot to run folder
5. Build normalized agent payload
6. Send payload to Codex
7. Send payload to Claude
8. Save Codex output
9. Save Claude output
10. Write run metadata
11. End in review state

## Workflow ownership rules
- `run_id` is generated by the workflow at trigger time, preferably as a UUID or timestamp-plus-random suffix.
- If a run folder already exists for the generated `run_id`, the workflow should fail fast rather than overwrite artifacts.
- `output_root` is the caller-supplied base directory. The workflow assembles the final run folder and file paths beneath it.

## Input payload
```json
{
  "task_id": "task-001",
  "project_id": "toc",
  "source_document_path": "docs/example.md",
  "goal": "Improve clarity and structure",
  "constraints": [
    "Preserve meaning",
    "Do not remove major sections"
  ],
  "output_root": "runs"
}
```

## Normalized agent payload
```json
{
  "task": "document_edit",
  "project_id": "toc",
  "goal": "Improve clarity and structure",
  "constraints": [
    "Preserve meaning",
    "Do not remove major sections"
  ],
  "document_text": "<loaded file contents>",
  "output_format": {
    "summary": "string",
    "proposed_text": "string",
    "change_notes": ["string"],
    "open_questions": ["string"]
  }
}
```

The workflow should add the generated `run_id` internally before saving artifacts or issuing agent calls.

## Run folder layout
- `runs/<run_id>/input.md`
- `runs/<run_id>/codex-output.md`
- `runs/<run_id>/claude-output.md`
- `runs/<run_id>/run.json`

## Success conditions
- source loads
- both agent responses include `summary`, `proposed_text`, `change_notes`, and `open_questions`
- both `proposed_text` fields are non-empty
- both outputs save successfully
- run metadata exists
- run metadata includes valid `started_at` and `ended_at`
- duplicate `run_id` collisions fail rather than overwrite
- source file remains unchanged
