Ecosystem Overview
Everything below feeds into each other. The TOC is the map. The G3 is the advisor. The goal: no project dies from neglect — only from deliberate decision.
The Four Pillars
The Brain
Four Claude instances (Claudio, Claude, Claudius, Clodius) covering browser, mobile, desktop, and API automation. Each has different connectors.
→ Claudius for deep work. Clodius for automation. Claude mobile for quick capture. Claudio for web research.
The Memory
Notion (structured, queryable), Google Drive (documents, files), Obsidian (local linked lore). All three serve different purposes — none replaces the others.
→ Session closeouts to Notion. Long-form writing to Drive. Lore and world-building to Obsidian.
The Muscle
n8n (automation workflows), RunPod (GPU compute), VPS (always-on hosting), local machine (UE5, Blender, Obsidian). Tools fire when triggered by n8n.
→ n8n is the connective tissue. Set up VPS first so routines run without your machine.
The Output
Games (Overcast Studios / UE5), Companion (Astrid), Stories (Atreal/Aedaeion), Writing (Essays/AUSA), TV Channel (KPTV-12 replica), RPG campaigns.
→ Every project has a tangible artifact at the end. If it doesn't, it's research, not a project.
Recommended Priority Order
Tier 1 // Unlock
n8n Foundation, Calendar Mission Setup, Claude Ecosystem Map. These unblock everything else. Do these first.
Tier 2 // Build
Overcast Studios setup, Astrid DB, TIE Fighter Academy, AI GM System. Core projects that require the Tier 1 infra.
Tier 3 // Create
Essays, Atreal novel, Aedaeion world, 5 Parsecs, SW Ship Game, TV Channel. High-reward creative work. Best after infra is stable.
Tier 4 // Sustain
Home/property tasks, finance hygiene, Google Calendar maintenance. Monthly/seasonal cadence. Low friction but high consequence if ignored.
The Key Insight

You have a single-operator TOC running a full multi-domain portfolio. The risk isn't capability — it's context-switching cost. The G3 picker and Notion closeout logs solve this: you never have to remember where you left off, because every session writes its state to Notion, and every new session reads it back. The calendar handles the scheduling pressure. The G3 handles the decision fatigue of "what do I work on tonight." Your only job is to execute the steps and log the closeout.
